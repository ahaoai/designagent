# 02-Discover：调研与定义

## 概述
第二步：发散进入调研。理解用户、上下文、竞争者和约束，然后收敛为书面设计简报。这结合了调研（原 design-discover）与综合成问题定义（原 design-define）。目标是先获得洞察，然后是清晰的简报。

## 不可协商的规则
**硬性门槛：在生成任何概念之前，必须完成调研综合和书面设计简报。** "我已经了解问题"不是调研。在任何视觉探索开始之前，设计简报必须存在。

## 何时使用
- 导入完成后（01-intake 已完成）
- 任何概念生成或视觉探索之前

## 何时不使用
- 导入期间（你知道的还不够——先完成 01）
- 用户要求快速草图（路由到精简模式）
- 所有调研已经存在且只需被引用（记录它然后进入 03）

## 流程

### 阶段A：调研（发散）

#### 1. 利益相关者和用户研究
- 主要和次要用户是谁？
- 他们的目标、痛点和行为是什么？
- 使用场景：在哪里、什么时候、怎么用、用什么设备/空间？
- 当前工作流或体验（如果是再设计）

#### 2. 上下文和竞争审计
- 现有品牌资产、设计系统、指南
- 竞争格局：直接竞争者、标杆参照、跨行业灵感
- 文化背景和市场环境
- 平台/媒介约束

#### 3. 收集灵感和参考
- 收集视觉参考、先例、案例研究
- 如需视觉研究，调用 **visual-research**
- 记录什么有效、什么无效

### 阶段B：定义（收敛）

#### 4. 综合调研
- 将发现聚类为主题
- 识别矛盾和张力
- 记录需要验证的未知项
- 产出一段调研摘要

#### 5. 撰写设计简报
- 问题陈述（我们在解决什么？）
- 项目范围和约束
- 目标受众和关键洞察
- 成功标准（从导入而来，经过细化）
- 设计原则（3-5条指导规则）
- 必须包含：范围之内且范围之外

## 理性化预防

| 借口 | 事实 |
|------|------|
| "这只是个小项目，不需要调研" | 每个设计都生活在上下文中。跳过调研等于闭眼设计。 |
| "我已经了解这个行业了" | 每个项目都是独特的。不要做任何假设。 |
| "我边调研边画草图" | 没有调研的草图是装饰，不是设计。 |
| "简报很明显" | 不管怎样写下来。明显的简报隐藏最多的假设。 |

## 危险信号
- 你在简报写出来之前就在生成视觉想法
- 你还没有向用户提问澄清问题
- "简报"是一段没有约束或范围说明的段落
- 你跳过了竞争格局分析

## 验证
- [ ] 调研摘要已产出（至少一段）
- [ ] 至少记录了3个用户需求/痛点
- [ ] 至少映射了2个竞争者
- [ ] 设计简报已撰写，包含问题陈述、范围、受众、标准、原则
- [ ] 范围之内和范围之外明确列出
- [ ] 尚未生成任何设计解决方案
