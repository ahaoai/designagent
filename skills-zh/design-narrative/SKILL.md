# 设计叙事（Design Narrative）

## 不可协商的规则
**硬性门槛：从问题开始，不是解决方案。** 在展示作品之前先呈现研究和理由。每个设计选择必须追溯到策略原因。不允许"我喜欢它"作为辩护。

## 概述
设计不会自己卖自己——它需要一个故事。此技能管理如何构建设计演示、围绕创意决策构建叙事，并用理由而非"我喜欢它"来为作品辩护。好的叙事将主观偏好转化为客观论证。

## 何时使用
- 向客户或利益相关者展示概念
- 撰写案例研究或作品集条目
- 在评审中辩护设计决策
- 创建设计提案或竞标
- 为交付记录设计理由

## 何时不使用
- "我怎么演示这个？"作为模糊问题 → 当你有内容要结构化时使用
- 不需要叙事结构的内部工作会议
- 快速Slack更新（这是给正式演示和文档的）

## 设计故事结构

### 1. 从问题开始，不是解决方案
- "这是我们学到的关于[用户/上下文/挑战]的内容"
- 框定利害关系：如果这个问题没解决好会怎样？
- 让观众在看到答案之前先感受到问题

### 2. 展示旅程
- "我们探索了[X]个方向，因为[理由]"
- 展示思考过程，不仅是结果
- 包含你尝试过但放弃了什么——它证明严谨性

### 3. 将解决方案呈现为水到渠成
- "给定[研究发现]和[约束]和[目标]，这个方向自然随之而来"
- 每个视觉选择都应该追溯到一个策略理由
- 色彩选择→与品牌人格或用户需求绑定
- 字体选择→与语调或可读性需求绑定
- 布局选择→与用户行为或内容层级绑定

### 4. 预判异议
- 预先应对明显的问题
- "您可能会想为什么我们没有[替代方案]——原因如下"
- 这建立信任：你已经比他们想得更深了

### 5. 以下一步结束，不是以问题结束
- 不要以"你觉得怎么样？"结束——那会邀请主观意见
- 以"这是我们的建议及理由。下一步：[具体行动]"结束
- 你是专家。引领。

## 三层理由系统
对每个设计决策，准备在三个层面上辩护：

| 层面 | 问题 | 示例回答 |
|------|------|---------|
| **策略** | 为什么这对业务/目标重要？ | "这个色彩系统让我们与品类中的每个竞争者区分开来。" |
| **功能** | 为什么这对用户有效？ | "高对比度确保了我们老龄人口统计的可读性。" |
| **审美** | 为什么这感觉对？ | "圆润的形态呼应了品牌的温暖和亲和力。" |

薄弱的演示只在审美层面操作。强有力的演示以策略引领。

## 案例研究格式
用于作品集和文档：
1. **背景**：客户、挑战、约束、时间线
2. **问题**：需要解决什么（具体、可衡量）
3. **过程**：调研→发想→开发（展示工作过程）
4. **解决方案**：最终设计（视觉优先，解释其次）
5. **影响**：发生了什么？（指标、证言、前后对比）
6. **反思**：你会有什么不同的做法？

## 理性化预防

| 借口 | 事实 |
|------|------|
| "作品自己会说话" | 它不会。未被解释的好设计只是装饰。 |
| "我就带他们过一遍界面" | 利益相关者需要叙事，不是导览。讲好故事，否则他们会自己写一个。 |
| "我不需要解释每个选择" | 未解释的选择成为主观修改的靶子。 |

## 危险信号
- 你的演示从LOGO开始
- 你无法为一个关键设计选择回答"为什么"
- 你对未预先处理的异议感到紧张
- 故事只是"这是我做的"



## 设计伦理与负责任叙事

### 设计伦理清单
- [ ] **代表性**：视觉是否真实且尊重地呈现目标受众？
- [ ] **无障碍**：残障人士能否感知、理解和参与？
- [ ] **暗黑模式**：是否存在操纵性或欺骗性的交互模式？
- [ ] **环境影响**：设计选择（印刷量、数据负载、材料规格）是否考虑可持续性？
- [ ] **文化挪用**：文化符号是否在理解和许可下使用，而非随意装饰？
- [ ] **数据隐私**：如果设计涉及用户数据，同意是否明确、控制权是否在用户手中？
- [ ] **包容性语言**：文案是否不含与性别、种族、年龄、能力或社会经济状态相关的偏见？
- [ ] **算法透明**：如果设计中涉及AI/ML系统，用户能否理解决策是如何做出的？

### 原则
- 设计伦理不是最后的勾选框——它贯穿每个阶段。
- 当任何阶段出现伦理关切时，暂停并提出。最好的叙事承认艰难的选择。
- 如果你必须在伦理维度上妥协，请明确记录这个权衡。

## 验证
- [ ] 问题在解决方案之前陈述
- [ ] 每个主要设计选择都有策略理由
- [ ] 至少展示了一个替代方案并解释了为什么不选
- [ ] 常见异议已预先处理
- [ ] 演示以具体的下一步结束，而非开放式问题
- [ ] 设计伦理清单已审查
