# 建筑设计（Architecture Design）

## 不可协商的规则
**硬性门槛：场地和功能分析必须先于形式生成。建筑必须回应其上下文——物理的、文化的和环境的。没有场地理解就没有体量设计。**

## 概述
建筑的领域专属方法论层。建筑是在建筑环境尺度上塑造人类体验的建筑和结构设计。此技能叠加在DesignAgent 线性流程上（01-intake → 02-discover → 03-strategy → 04-generate → 05-review → 06-deliver → 07-learn），在每个阶段注入建筑专属框架。

## 何时使用
- 建筑设计（住宅、商业、市政、文化、机构）
- 城市设计和总体规划
- 适应性再利用和翻新
- 景观和环境设计
- 结构、围护系统和设备设计
- 可行性研究和概念设计

## 何时不使用
- "画个房子立面" → 快速草图，用精简模式
- 仅室内项目（路由到 interior-design）
- 结构工程计算（这是设计方法论，不是工程）
- 单房间加建，不需要场地分析

## 建筑专属流程

### 阶段1：场地和上下文（叠加01-intake → 02-discover）
超越一般调研进入建筑领域：
- **场地分析**：朝向、日照路径、风模式、地形、土壤、植被
- **上下文分析**：周边建筑、街道模式、城市肌理、视线
- **文化背景**：当地建筑传统、材料、类型、乡土建筑
- **法规背景**：区划、建筑规范、退界、限高、历史保护区
- **环境分析**：气候数据、能源建模、可持续性目标
- **功能分析**：空间需求、相邻关系、面积计算

### 阶段2：设计概念（叠加03-strategy）
在形式设计之前定义建筑意图：
- **设计概念**：驱动设计的核心想法（一句话）
- **帕提图解**：基本的组织原则或图示
- **体量策略**：体积、尺度和比例如何与上下文关联
- **材料和构造概念**：由什么制成以及如何建造？
- **环境策略**：被动设计、朝向、围护性能

### 阶段3：设计深化（叠加04-generate）
- **体量方案**：至少2个不同的形式配置
- **平面深化**：平面图、剖面、立面研究
- **剖面研究**：空间体验、光线、垂直动线
- **立面和围护**：材料性、开窗、节奏、比例
- **结构概念**：传力路径、结构系统、跨度
- **如需探索视觉风格**，调用 **designing-with-ai** 进行可视化

### 阶段4：评审与细化（叠加05-review）
- 对照功能需求审查：满足空间要求吗？
- 对照场地和上下文审查：是否做出了适当回应？
- 对照环境目标审查：能源、日照、舒适度
- 对照法规审查：区划、规范、无障碍
- 用户测试：人们如何在空间中体验？

### 阶段5：文档与交付（叠加06-deliver → 07-learn）
- 建筑图纸：平面、剖面、立面、节点、清单
- 规格：材料、系统、性能标准
- 与咨询方协调：结构、机电、景观、室内
- 对照原始功能需求、预算和时间线验证

## 理性化预防

| 借口 | 事实 |
|------|------|
| "我先做一个引人注目的形式，后面再解释" | 没有上下文的形式是雕塑，不是建筑。场地和功能优先。 |
| "可持续性可以后面再加" | 可持续性是设计驱动力，不是检查清单。从一开始就整合。 |
| "这块场地是一张白纸" | 没有场地是白纸。每块场地都有朝向、气候和上下文。 |
| "我设计的时候再处理功能需求" | 功能需求驱动设计。在体量设计之前定义它。 |

## 危险信号
- 你在分析场地之前就开始做体量
- 设计没有回应气候或朝向
- 你不了解区划或规范要求
- 帕提是"我看到的时候就知道了"
- 环境策略没有被考虑

## 验证
- [ ] 场地和上下文分析已完成
- [ ] 功能需求分析及面积要求已记录
- [ ] 设计概念和帕提图解已产出
- [ ] 至少生成了2个体量方案
- [ ] 环境策略已定义
- [ ] 平面、剖面和立面已开发
- [ ] 已对照功能需求、预算和法规要求验证
