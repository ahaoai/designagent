# 04-Generate：概念、原型与生成

## 概述
生成阶段：发散进入多个概念方向，原型化最强的方向，使用AI工具加速探索。这结合了概念生成（原design-ideate）、原型制作（原design-prototype）和AI辅助生成。目标是产出2-3个不同的、可测试的方向——而不是一个"正确"的答案。

## 不可协商的规则
**硬性门槛：在收敛到一个方向之前，至少生成3个不同的方向。** 一个方向不是一个选择。使用AI来扩展范围，但你来策展和决策。不要将原始AI输出作为最终概念展示。

## 流程

### 1. 概念生成（发散）
- 至少生成3个不同的视觉或概念方向
- 每个方向应有清晰的理由链接到策略
- 使用对比方法（例如，传统vs现代、大胆vs内敛）
- 如需探索视觉风格，调用 **designing-with-ai** 进行图像生成
- 如需探索视频或动效，调用 **designing-with-video**

### 2. 概念矩阵
创建一个跨方向的比较矩阵：

| 标准 | 方向A | 方向B | 方向C |
|------|------|------|------|
| 符合策略？ | | | |
| 可行性 | | | |
| 差异化 | | | |
| 风险 | | | |
| 用户反应 | | | |

### 3. 原型和测试
- 为1-2个最强方向构建低保真原型
- 每个原型回答一个具体问题
- 如有可能，与用户或利益相关者测试
- 记录：什么有效、什么困惑、他们希望什么
- 基于反馈迭代（在此阶段内循环）

### 4. 收敛
- 选择最强的方向（或混合方案）
- 记录为什么选择它、拒绝了什么
- 细化到中高保真
- 如使用了AI：确保输出已被策展和细化，非原始输出

### 5. 准备评审
- 展示带有方向理由的概念
- 展示什么被探索和拒绝了
- 如需演示协助，调用 **design-narrative**

## 何时使用
- 策略获批后（03-strategy 完成）
- 准备探索多个不同的视觉或概念方向时

## 何时不使用
- 调研或简报撰写期间（回到 02 或 03）
- 用户只想要对已有作品的评审（使用 05-review）
- "就把 LOGO 改这一处" → 这是修改，不是生成
- 策略标准还没定义（你会盲人摸象）

## 理性化预防

| 借口 | 事实 |
|------|------|
| "一个强方向就够了" | 一个方向是猜测。三个方向是决策。 |
| "我用AI做最终概念" | AI生成选项。你策展、细化和决定。 |
| "原型制作太花时间" | 原型揭示思考无法揭示的东西。低成本失败。 |
| "我已经知道哪个是最好的" | 测试它。你的直觉需要证据。 |
| "其他方向只是凑数" | 如果你无法生成3个真正不同的方法，你就还没理解问题。 |
| "我给用户看一个方向然后从这里迭代" | 一个方向是最后通牒。三个方向是对话。 |

## 危险信号
- 你只展示一个方向
- AI输出是原始展示的（未策展、未细化）
- 概念矩阵填写为空或模糊
- 你无法表达为什么一个方向更强
- 原型被跳过

## 验证
- [ ] 至少生成了3个不同方向
- [ ] 概念矩阵已完成并记录
- [ ] 至少构建并测试了1个原型
- [ ] 反馈已记录并应用
- [ ] 所选方向记录了理由
- [ ] AI输出（如使用）已策展和细化

→ 下一步：使用 **05-review** 进行评审、收集反馈并在定稿前验证。

## 设计决策日志（重要决策必填）

所有重要的生成决策必须记录。这里记录"为什么不选别的"推理，使后续审计成为可能。

**何时记录：**
- 从概念矩阵中选定最终方向
- 决定融合两个方向而非单选一个
- 任何改变方向特征的重大细化选择
- 任何 AI 生成决策：生成了什么、保留了什么、丢弃了什么、为什么

**日志格式（与 03-strategy 相同）：**

```
决策：[决定了什么]

选项：
A. [方向/方法——命名，简要描述]
B. [方向/方法——命名，简要描述]
C. [选定方案——命名，简要描述]

选定：C

拒绝：
A → [为什么——与 03-strategy 的策略标准挂钩]
B → [为什么——与 03-strategy 的策略标准挂钩]

理由：
→ [为什么C最突出——与评估标准、用户反馈、可行性挂钩]
```

**规则：**
- 没有记录拒绝理由的方向选择是空洞的决策。记录下来。
- 如果在生成中使用了 AI：记录使用了什么提示词、保留了什么输出、丢弃了什么。
- 日志条目是 07-learn 审计中的证据。审计将交叉检查：选定的方向是否追溯至策略标准，拒绝理由是否与决策日志一致？
- 精简模式：仅记录方向选择决策。标准/深度模式：记录所有重要的生成决策。
